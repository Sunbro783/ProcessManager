ProcessManager — это настольное приложение для операционной системы Windows, предназначенное для мониторинга и управления запущенными процессами. Программа разработана на языке C# с использованием технологии WPF и архитектурного шаблона MVVM. Основная цель проекта — предоставить пользователю удобный интерфейс для анализа состояния системы и управления процессами в реальном времени.

Приложение отображает список всех запущенных процессов с основными характеристиками, включая идентификатор, имя, приоритет, объем используемой памяти, процессорное время и количество потоков. Реализована возможность фильтрации процессов по имени, отображения только графических процессов, а также выделения системных процессов. Список автоматически обновляется через настраиваемый интервал времени.

Пользователь может завершить выбранный процесс или изменить его приоритет выполнения. Для предотвращения нестабильности системы предусмотрено предупреждение при выборе приоритета реального времени. Дополнительно реализован просмотр потоков выбранного процесса с отображением их идентификаторов, приоритета, состояния и затраченного процессорного времени.

В приложении предусмотрено древовидное отображение процессов, позволяющее видеть иерархию родительских и дочерних процессов. Это облегчает анализ структуры запущенных приложений и служб.

Отдельное внимание уделено управлению CPU Affinity. Пользователь может выбрать процесс и указать, на каких логических ядрах процессора ему разрешено выполняться. Изменения применяются немедленно и сохраняются при обновлении списка процессов. Если процесс не выбран, элементы управления CPU Affinity недоступны.

Для визуализации состояния системы реализованы графические компоненты. Столбчатая диаграмма отображает загрузку каждого логического ядра процессора в процентах. Значения выводятся во всплывающих подсказках при наведении курсора. Круговая диаграмма отображает десять процессов с наибольшим потреблением оперативной памяти. В секторах отображается объем памяти в мегабайтах, а название процесса выводится во всплывающей подсказке.

Проект структурирован в соответствии с принципами MVVM и разделен на папки Models, Services, ViewModels и Converters. Модели описывают структуры данных процессов, потоков и ядер процессора. Сервисы отвечают за взаимодействие с операционной системой и получение системной информации. ViewModel содержит основную бизнес-логику приложения и связывает пользовательский интерфейс с данными. Конвертеры используются для отображения данных в интерфейсе.

В работе используются библиотеки System.Diagnostics для взаимодействия с процессами операционной системы, Windows Management Instrumentation для получения информации о процессоре и библиотека LiveCharts для построения графиков.
